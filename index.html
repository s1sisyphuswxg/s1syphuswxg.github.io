Docker + Selenium Grid构建Web测试环境

1. 基础知识

Selenium
* 是针对Web应用的自动化测试框架和工具集合
* 支持多种浏览器和编程语言
* 测试用例直接运行在浏览器中，并模拟用户的操作

Selenium Grid
* 是一种可以进行分布式自动化测试的辅助工具，该架构中包含两个主要角色
* Hub是中心点控制节点
* Node是Selenium的工作节点，它们注册到Hub上，并会操作浏览器执行由Hub下发的自动测试用例

Selenium官方已经支持了Docker搭建镜像环境，地址：https://hub.docker.com/r/selenium/

构建selenium的Dockerfile来自Github：https://github.com/SeleniumHQ/docker-selenium

2. 实践

1). 拉取最新镜像

docker pull selenium/hub
docker pull node-chrome-debug

2). 启动容器

docker run -d -p 4444:4444 --name selenium-hub selenium/hub
docker run -d -P -p 5901:5900 --link selenium-hub:hub --name chrome-node1 selenium/node-chrome-debug

3). 检查容器是否正常启动

 ~/ docker ps | grep selenium
f262c6ef947c        selenium/node-chrome-debug   "/opt/bin/entry_po..."   6 seconds ago       Up 4 seconds        0.0.0.0:5901->5900/tcp   chrome-node1
e47bee85df3d        selenium/hub                 "/opt/bin/entry_po..."   2 minutes ago       Up 2 minutes        0.0.0.0:4444->4444/tcp   selenium-hub

4). 浏览器中输入[HOST_IP]:4444/grid/console可以查看节点注册情况
节点如果为debug，可以通过vnc连接[HOST_IP]:5901可以远程连接到node1（密码为secret) ;
也可以通过 Safari中 vnc://[HOST_IP]:PORT 中查看。

至此，Docker + Selenium 测试环境已经搭建OK。
